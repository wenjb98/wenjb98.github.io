设计模式--单例

设计模式——单例

1.特点

  • 整个程序中必须仅存在唯一一个单例对象
  • 单例类中需定义本类的对象私有静态属性，私有化构造方法
  • 必须提供一个给外部获取实例的方法

2.懒汉式

2.1线程不同步（线程不安全）

​ 这是实现单例设计模式的方法之一，但是此方法尤其重大缺陷

public class Singleton1 {

    public   int a = 1;

    private static Singleton1 instance  = null ;

    private Singleton1(){

    }
    public static Singleton1 getInstance(){

        if (instance == null){  //线程二
             instance = new Singleton1();  //线程一
        }
        return  instance;
    }
  }

​ 若是不仔细推敲一番，很难发现其问题所在，但是将代码带入多线程的环境下就会发现，在线程一执行对象初始化语句后，若是被线程二抢占执行权，此时的instance依旧为null,就会出现多个线程执行对象初始化，并且赋值

单例也就不攻自破。

2.2线程同步

同过上面的问题，此时首先想到的可能就是使用synchronized，来同步getInstance方法。

public static synchronized Singleton1 getInstanceSyn(){

      if (instance == null){
          instance = new Singleton1();
      }
      return  instance;
  }

这种方法确实能解决出现多实例的bug，但是这样并不高效，因为在任何时候调用此方法，都需要承受线程同步所带来的开销。

这时候就需要引入DCL 双重检查锁模式

public static synchronized Singleton1 getInstanceDCLP(){

    if (instance == null){
        synchronized (Singleton1.class){
            if (instance == null){
                instance = new Singleton1();
            }

        }
    }
    return  instance;
}

这种方法用了两次条件判断instance是否为空，在第一次判断的时候不加锁，第二次加锁，这样当对象被实例化之后，将不会再去执行同步代码块中的内容，也就不需要承担同步所带来的资源开销。

3.饿汉式

​ 这种方式就比较简单粗暴，同事也说使用较多的一种实现单例的方法，在程序加载之后就立即初始化私有对象属性。

/**
 * 饿汉单例模式
 */
public class Singleton2 {
    
    private  static Singleton2  instance= null;
    
    private Singleton2(){
        
    }
    static {
        instance = new Singleton2();
    }

    public static Singleton2 getInstance() {
        return instance;
    }
}

这种模式是天生线程安全的，但是在系统项目启动后，不管你时候调用获取实例的方法，都会消耗内存空间。
